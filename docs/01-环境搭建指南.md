## 2. Python环境搭建

### 2.1 安装Python 3.11+

> **什么是Python**：一种广泛使用的编程语言，在AI/ML领域是最主流的选择。本项目的Python版使用了LangGraph、FastAPI等流行框架。

**macOS（推荐使用Homebrew）**：

```bash
# 安装Homebrew（如果还没有）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装Python
brew install python@3.11

# 验证
python3 --version
# 输出应该类似：Python 3.11.x 或更高版本
```

**Windows**：

```bash
# 方法1：从官网下载安装包
# 访问 https://www.python.org/downloads/ 下载 Python 3.11+ 安装包
# 安装时一定要勾选 "Add Python to PATH" !!!

# 方法2：使用winget
winget install Python.Python.3.11

# 验证
python --version
```

**Linux（Ubuntu/Debian）**：

```bash
sudo apt update
sudo apt install python3.11 python3.11-venv python3-pip
python3.11 --version
```

### 2.2 创建虚拟环境

> **什么是虚拟环境（venv）**：虚拟环境就像一个"隔离的房间"，让这个项目的依赖不会跟你电脑上其他Python项目的依赖互相冲突。每个虚拟环境有自己独立的包安装目录。

```bash
# 进入Python项目目录
cd python

# 创建虚拟环境（会生成一个venv文件夹）
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate     # macOS / Linux
# venv\Scripts\activate      # Windows CMD
# venv\Scripts\Activate.ps1  # Windows PowerShell

# 激活后你会看到命令行前面多了 (venv) 标记
# (venv) $ ← 表示虚拟环境已激活
```

### 2.3 安装依赖

```bash
# 确保你在 python/ 目录下，且虚拟环境已激活
pip install -r requirements.txt
```

`requirements.txt` 中包含以下核心依赖：

| 包名 | 用途 | 说明 |
|------|------|------|
| `langgraph>=0.2` | Agent有向图编排框架 | 本项目的核心：定义5个Agent的流水线 |
| `langchain-openai>=0.2` | OpenAI LLM集成 | 调用GPT-4o-mini进行推理和生成 |
| `langchain-core>=0.3` | LangChain核心库 | 提供消息类型（SystemMessage、HumanMessage）等基础抽象 |
| `fastapi>=0.115` | Web框架 | 提供REST API服务 |
| `uvicorn>=0.30` | ASGI服务器 | 运行FastAPI应用的HTTP服务器 |
| `pydantic>=2.0` | 数据验证 | 定义PatientInfo、ClinicalState等数据模型 |
| `pydantic-settings>=2.0` | 配置管理 | 从.env文件自动加载环境变量 |
| `structlog>=24.4` | 结构化日志 | 输出JSON格式的日志，方便生产环境分析 |
| `httpx>=0.27` | HTTP客户端 | 调用外部FHIR服务器 |
| `neo4j>=5.25` | Neo4j驱动 | 连接知识图谱数据库（可选） |
| `pytest>=8.0` | 测试框架 | 运行单元测试 |
| `python-dotenv>=1.0` | .env文件加载 | 读取.env配置文件 |

### 2.4 pip常见问题

```bash
# 如果pip install很慢，可以使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 如果遇到权限问题
pip install --user -r requirements.txt

# 如果pip版本太旧
pip install --upgrade pip
```

## 3. Java环境搭建

### 3.1 安装JDK 17+

> **什么是JDK**：Java Development Kit，Java开发工具包。JDK 17是长期支持版本（LTS），Spring Boot 3.x 要求最低JDK 17。

**macOS**：

```bash
brew install openjdk@17

# 设置JAVA_HOME
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 17)' >> ~/.zshrc
source ~/.zshrc

# 验证
java -version
# 输出应该类似：openjdk version "17.0.x"
```

**Windows**：

```bash
# 推荐从Adoptium下载：https://adoptium.net/
# 选择 JDK 17 LTS，下载安装

# 设置环境变量JAVA_HOME指向JDK安装目录
# 将 %JAVA_HOME%\bin 添加到PATH

java -version
```

**Linux**：

```bash
sudo apt install openjdk-17-jdk
java -version
```

### 3.2 安装Maven

> **什么是Maven**：Java世界的包管理和构建工具，类似于Python的pip + setuptools。它通过 `pom.xml` 文件管理项目依赖。

```bash
# macOS
brew install maven

# Linux
sudo apt install maven

# 验证
mvn -version
# 输出应该显示 Maven 3.9.x 和 Java 17
```

### 3.3 编译Java项目

```bash
cd java

# 下载依赖并编译（跳过测试，加快速度）
mvn clean package -DskipTests

# 编译成功后会在 target/ 目录生成JAR文件
ls target/*.jar
```

Java版核心依赖（`pom.xml`中定义）：

| 依赖 | 用途 |
|------|------|
| Spring Boot 3.3 | Web框架 + 依赖注入 |
| Spring AI 1.0 | LLM调用集成 |
| LangGraph4j 1.8 | Agent编排（参考） |
| Lombok | 代码生成（@Data、@Builder） |
| Jackson | JSON序列化/反序列化 |

## 5. Docker环境搭建

### 5.1 安装Docker

> **什么是Docker**：一个容器化工具，可以把软件和它的依赖打包成一个"容器"运行。你不需要在自己电脑上安装PostgreSQL、Neo4j、Redis——Docker帮你一键搞定。

**macOS**：

```bash
# 方法1：下载Docker Desktop
# 访问 https://www.docker.com/products/docker-desktop/ 下载安装

# 方法2：使用Homebrew
brew install --cask docker
# 安装后需要打开Docker Desktop应用
```

**Windows**：

- 从 https://www.docker.com/products/docker-desktop/ 下载安装Docker Desktop
- 安装时会提示开启WSL 2（Windows Subsystem for Linux），按提示操作即可

**Linux**：

```bash
# Ubuntu
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER
# 重新登录后生效
```

### 5.2 验证Docker安装

```bash
docker --version
# Docker version 24.x.x

docker compose version
# Docker Compose version v2.x.x

# 测试运行
docker run hello-world
```

### 5.3 启动基础设施服务

本项目需要三个基础设施服务，通过docker-compose一键启动：

| 服务 | 端口 | 用途 | 控制台地址 |
|------|------|------|-----------|
| **PostgreSQL** | 5432 | 审计日志存储、会话记录 | — |
| **Neo4j** | 7474/7687 | 医学知识图谱（GraphRAG） | http://localhost:7474 |
| **Redis** | 6379 | 缓存、会话管理 | — |

```bash
# 在python/目录下（或go/目录下，都有docker-compose.yml）
cd python

# 启动所有服务（后台运行）
docker compose up -d

# 查看运行状态
docker compose ps
```

**期望输出**：

```
NAME         STATUS              PORTS
postgres     running (healthy)   0.0.0.0:5432->5432/tcp
neo4j        running             0.0.0.0:7474->7474/tcp, 0.0.0.0:7687->7687/tcp
redis        running (healthy)   0.0.0.0:6379->6379/tcp
```

> **小白提示**：如果某个服务状态不是 `running`，可以用 `docker compose logs <服务名>` 查看日志排查原因。

### 5.4 验证数据库初始化

PostgreSQL会自动执行 `docker/init-db.sql`，创建以下表：

```bash
# 连接到PostgreSQL验证
docker exec -it postgres psql -U postgres -d clinical_decision -c "\dt"

# 应该看到两张表：
#  audit_logs          | 审计日志表（HIPAA要求，不可删除）
#  clinical_sessions   | Pipeline会话记录
```

## 6. 配置环境变量（.env文件详解）

### 6.1 创建.env文件

```bash
cd python
cp .env.example .env
```

### 6.2 .env文件字段详解

```bash
# ============================================
# LLM 配置（最重要！）
# ============================================

# 你的OpenAI API密钥（必须）
# 格式：sk-开头的一串字符
OPENAI_API_KEY=sk-your-api-key-here

# 使用的LLM模型（可选，默认gpt-4o-mini）
# gpt-4o-mini: 最便宜，速度最快，对本项目完全够用
# gpt-4o: 更强大，适合处理复杂/罕见病例
# gpt-4-turbo: 最贵，一般不需要
OPENAI_MODEL=gpt-4o-mini

# ============================================
# PostgreSQL 配置
# ============================================

# PostgreSQL主机地址（Docker运行时用localhost）
POSTGRES_HOST=localhost
POSTGRES_PORT=5432

# 数据库名（Docker会自动创建）
POSTGRES_DB=clinical_decision

# 数据库用户名和密码
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres    # 生产环境请使用强密码！

# ============================================
# Neo4j 知识图谱配置
# ============================================

# Neo4j连接地址
# bolt:// 是Neo4j的二进制协议，7687是默认端口
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=neo4jpass       # 生产环境请使用强密码！

# ============================================
# Redis 缓存配置
# ============================================

REDIS_HOST=localhost
REDIS_PORT=6379

# ============================================
# FHIR 服务配置（可选）
# ============================================

# 外部FHIR服务器地址（如HAPI FHIR Server）
# 不配置也不影响核心功能
FHIR_SERVER_URL=http://localhost:8080/fhir

# ============================================
# 应用配置
# ============================================

# API服务监听地址和端口
APP_HOST=0.0.0.0     # 0.0.0.0 表示监听所有网络接口
APP_PORT=8000         # Python版默认8000

# 日志级别：DEBUG / INFO / WARNING / ERROR
LOG_LEVEL=INFO
```

### 6.3 关键配置项说明

| 配置项 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `OPENAI_API_KEY` | ✅ 必须 | 无 | 没有这个Key，LLM相关功能（4个Agent）无法工作 |
| `OPENAI_MODEL` | 否 | `gpt-4o-mini` | 推荐使用默认值，性价比最高 |
| `POSTGRES_PASSWORD` | ⚠️ 建议修改 | `postgres` | 生产环境必须使用强密码 |
| `NEO4J_PASSWORD` | ⚠️ 建议修改 | `neo4jpass` | 生产环境必须使用强密码 |
| 其他 | 否 | 见上方 | Docker环境下使用默认值即可 |

## 7. OpenAI API Key获取步骤

> **什么是API Key**：调用OpenAI GPT模型的"通行证"。每次你的Agent调用LLM，都需要携带这个Key来验证身份和计费。

### 7.1 注册账号

1. 访问 [platform.openai.com](https://platform.openai.com)
2. 点击 "Sign up" 注册账号（支持Google/Microsoft/Apple账号登录）
3. 验证邮箱和手机号

### 7.2 创建API Key

1. 登录后访问 [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
2. 点击 "Create new secret key"
3. 给Key起一个名字（如 "clinical-decision-system"）
4. 复制生成的Key（**只会显示一次！**请立即保存）

### 7.3 费用说明

| 模型 | 输入价格 | 输出价格 | 单次Pipeline成本 | 备注 |
|------|---------|---------|-----------------|------|
| gpt-4o-mini | $0.15/百万token | $0.60/百万token | ~$0.002 | **推荐**，$1可跑500次 |
| gpt-4o | $2.50/百万token | $10.00/百万token | ~$0.03 | 更强但贵17倍 |
| gpt-4-turbo | $10.00/百万token | $30.00/百万token | ~$0.10 | 最贵，一般不需要 |

> **小白提示**：新账号通常有$5免费额度。使用gpt-4o-mini的话，$5可以跑大约2500次完整Pipeline，足够开发和面试演示使用。

### 7.4 替代方案

如果无法使用OpenAI API，有以下替代方案：

| 方案 | 配置方式 | 说明 |
|------|---------|------|
| **通义千问**（阿里云） | `OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1`，模型用`qwen-plus` | 国产大模型，兼容OpenAI接口 |
| **智谱AI** | `OPENAI_BASE_URL=https://open.bigmodel.cn/api/paas/v4`，模型用`glm-4-flash` | 国产大模型 |
| **Ollama本地** | `OPENAI_BASE_URL=http://localhost:11434/v1`，模型用`llama3` | 完全本地运行，无需API Key |
| **无LLM模式** | 不设置API Key | ICD-10搜索、DDI检查等不依赖LLM的功能仍可使用 |

## 9. 常见问题排查

### 9.1 Docker相关问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `docker: command not found` | Docker未安装 | 安装Docker Desktop |
| `Cannot connect to Docker daemon` | Docker服务未启动 | 打开Docker Desktop应用 |
| 端口5432被占用 | 本地已有PostgreSQL | `lsof -i :5432` 找到进程并停止，或修改docker-compose端口 |
| 端口7474被占用 | 本地已有Neo4j | 类似上一条 |
| Neo4j启动失败 | 内存不足 | Neo4j默认需要1GB内存，确保机器有足够空闲内存 |

**端口冲突排查通用命令**：

```bash
# 查看端口占用
lsof -i :5432    # PostgreSQL
lsof -i :7474    # Neo4j Web控制台
lsof -i :7687    # Neo4j Bolt协议
lsof -i :6379    # Redis

# 如果需要释放端口
kill -9 <PID号>

# 完全清除Docker数据重新开始
docker compose down -v      # -v 会删除数据卷
docker compose up -d        # 重新创建
```

### 9.2 Python相关问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `ModuleNotFoundError: No module named 'langgraph'` | 依赖未安装 | `pip install -r requirements.txt` |
| `python3: command not found` | Python未安装或PATH未配置 | 安装Python并确认PATH |
| 虚拟环境激活失败 | 使用了错误的激活命令 | macOS/Linux用`source venv/bin/activate`，Windows用`venv\Scripts\activate` |
| pip安装超时 | 网络问题 | 使用清华镜像：`pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple` |
| `pydantic.errors.PydanticImportError` | Pydantic版本冲突 | `pip install pydantic>=2.0 --force-reinstall` |

### 9.3 API Key相关问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `AuthenticationError: Invalid API Key` | Key格式错误或已过期 | 检查.env中的OPENAI_API_KEY是否正确 |
| `RateLimitError` | 调用频率超限 | 等待几秒后重试，或升级API套餐 |
| `InsufficientQuotaError` | 余额不足 | 充值或换用免费额度的新账号 |
| 返回乱码/非JSON | 模型不支持 | 确认OPENAI_MODEL设置正确 |

### 9.4 一键环境检查脚本

你可以在终端运行以下命令快速检查环境是否就绪：

```bash
echo "=== 环境检查 ==="
echo "Python: $(python3 --version 2>/dev/null || echo '未安装')"
echo "Java:   $(java -version 2>&1 | head -1 || echo '未安装')"
echo "Go:     $(go version 2>/dev/null || echo '未安装')"
echo "Docker: $(docker --version 2>/dev/null || echo '未安装')"
echo "Compose: $(docker compose version 2>/dev/null || echo '未安装')"
echo ""
echo "=== Docker服务状态 ==="
docker compose ps 2>/dev/null || echo "Docker Compose未运行"
echo ""
echo "=== API Key检查 ==="
if [ -f .env ]; then
    grep -q "OPENAI_API_KEY=sk-" .env && echo "API Key: 已配置" || echo "API Key: 未配置或格式错误"
else
    echo ".env文件不存在，请先创建"
fi
```
